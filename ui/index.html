RAG Pipeline

1

Upload & Parse

Resume (PDF/DOCX) uploaded via UI. Text extracted using pdfplumber or python-docx. Raw file stored in MinIO.

2

LLM Extraction

llama3.1:8b via Ollama extracts structured JSON: name, email, skills, titles, years of experience, clearance level, education.

3

Chunk & Embed

Resume split into sections (summary, experience, skills, education). Each chunk embedded into a 768-dimensional vector using nomic-embed-text.

4

Store

Vectors stored in pgvector with HNSW cosine index. Metadata (skills, clearance, etc.) in PostgreSQL. Extracted JSON saved to MinIO.

5

Search & Retrieve

User query embedded with same model. Cosine similarity search against all resume chunks. Top 20 candidates retrieved with optional clearance/experience filters.

6

LLM Rerank & Explain

llama3.1:8b reranks top 20 to top 5, providing a one-sentence explanation per candidate. Chat mode generates conversational responses with candidate context.
